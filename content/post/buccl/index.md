## 설명
기존 프리다이빙 업체들은 네이버 카페와 스마트스토어를 중심으로 한 수기 예약·정산·커뮤니케이션 구조에 의존하고 있어, 수강생과 강사 모두에게 운영 비효율과 관리 부담이 발생하고 있었습니다. 이러한 아날로그 운영 방식을 DX(Digital Transformation) 관점에서 재정의하여, 강습·예약·결제·커뮤니티가 유기적으로 연결되는 버킷리스트클래스 플랫폼을 구축하였습니다.

‘나만의 버킷리스트, 함께하는 특별한 클래스’라는 서비스 비전 아래, 수강생은 클래스 탐색부터 예약, 참여 이후의 커뮤니티 활동까지 하나의 흐름으로 경험할 수 있도록 설계하였고, 강사는 수업 일정 관리, 수강생 관리, 정산을 디지털 환경에서 간편하게 운영할 수 있도록 시스템을 고도화하고 있습니다.

[여기](https://buccl.com/)에서 웹사이트를 확인할 수 있습니다.

## 담당 업무

### 서비스 전반 기획 · 디자인 · 프론트엔드 개발 리드
- 서비스 초기 단계부터 참여하여 전반적인 UX 기획, UI 설계, 프론트엔드 개발을 총괄
- 강습·투어·커뮤니티가 연결되는 플랫폼 구조 설계 및 사용자 흐름 정의

### 사용자 역할별 서비스 기능 구현
- 사용자 권한(Admin / 강사 / 사용자)에 따른 라우팅 및 접근 제어 구조 설계
- 권한별 메뉴, 화면 노출, 기능 접근을 일관되게 관리하는 프론트엔드 아키텍처 구축

**어드민(Admin)**
- 강습 및 자격증 승인 관리 기능 구현
- 제품 및 투어 등록·관리 화면 개발
- 플랫폼 운영을 위한 핵심 관리 기능 UI/UX 설계 및 구현

**강사(Instructor)**
- 회원 스케줄 관리 및 출석 관리 기능 개발
- 강습 운영에 필요한 정보 확인 및 관리 화면 제공

**사용자(User)**
- 강습 예약 시스템 구현
- 제품 및 수강권 결제 기능 개발
- 외부 결제 서비스 연동
- 커뮤니티 기능 및 콘텐츠 탐색 UI 구현
- 배송 정보 관리 및 주문 흐름 개발
- 앱 메인 페이지 UI 구성 및 사용자 경험 개선



## 어려웠던 점 / 배운점

프로덕트 초기 단계부터 참여하여 기획, 디자인, 프론트엔드 개발 전반을 책임지는 역할을 수행하면서 업무 범위와 책임이 크게 확장되었습니다. 단순히 요구사항을 구현하는 수준을 넘어, 백엔드 개발자 및 대표와 기술적 제약과 일정, 비즈니스 우선순위를 조율하며 의사결정을 이끌어야 했고, 이를 위해 논리적인 근거를 바탕으로 한 설득과 커뮤니케이션 역량이 지속적으로 요구되었습니다. 또한 배포 직후 실제 사용자가 유입되는 환경이었기 때문에, 개발 속도를 유지하면서도 안정성을 확보해야 하는 압박 속에서 잦은 운영 이슈 대응과 품질 관리까지 병행해야 했습니다.

이 과정에서 개발자 관점에 머무르지 않고, 사용자 경험과 운영 효율, 서비스 확장성을 포함한 프로덕트 전체를 바라보는 시야를 실무적으로 체득할 수 있었습니다. 프로젝트 리드로서 기술 선택과 일정 관리, 우선순위 판단을 주도하며 5년차 프론트엔드 개발자에게 요구되는 책임감과 주도성을 실제로 수행해본 경험이 되었고, 반복 작업과 설계·정리 과정에 AI 도구를 적극 도입하여 제한된 리소스 환경에서도 생산성을 높이는 방법을 익혔습니다. 이를 통해 안정적인 구현 역량을 넘어, 문제 정의부터 실행까지 책임지는 프론트엔드 리드 역할에 대한 이해와 경험을 축적할 수 있었습니다.
